# 块分配表

Fuyutsui 在屏幕顶端渲染**两行像素条**，将游戏状态编码后传递给 Python 端读取。

## 两行总览

| 行 | 帧名 | 位置 | 高度 | 槽位 | 用途 |
|---|---|---|---|---|---|
| 第一行（色块行） | `FuyutsuiColorBars` | TOPLEFT(0, 0) | 2px | 固定 255 个 | 状态块、光环、法术冷却、队伍信息 |
| 第二行（计数条行） | `FuyutsuiCountBars` | TOPLEFT(0, -2) | 20px | 动态分配 | 施法计数（castCount）、充能层数（charge） |

**关键源码：**
- `Fuyutsui/core/block.lua`：两行的帧创建和纹理写入
- `Fuyutsui/main.lua` 第 253–315 行 `loadPlayerBlocks()`：将职业定义编译为运行时结构
- `Fuyutsui/class/*.lua`：每个职业/专精的槽位定义
- `Fuyutsui/Fuyutsui/config.yml`：Python 端的 step 编号到字段名映射

---

## 第一行（色块行，colorBars）：255 个固定槽位

### 编码协议

每个像素颜色 `(R, G, B)` 编码含义：
- **R** = 0（标记为第一行色块，区别于第二行）
- **G** = `索引 / 255`（标识第几个槽位）
- **B** = `值 / 255`（0–1 浮点数，Python 读取为 0–255 整数）

索引 1 固定写为绿色 `(0, 1/255, 0)` 作为起始锚点，Python 端通过扫描绿色标记定位色块行起点。

### 槽位分配总览

255 个槽位分为几个区段：

| 区段 | 索引范围 | 类型 | 说明 |
|---|---|---|---|
| 全局状态块 | 1–20 | block | 所有职业/专精完全一致 |
| 职业专精块 | 21–约 30 | block | 职业特有的状态值 |
| 光环槽 | 约 21–60 | aura | 光环剩余时间/层数 |
| 法术冷却槽 | 约 30–73 | spell | 法术冷却（含充能法术的独立充能槽） |
| 物品/杂项块 | 约 46–53 | block | 药水冷却、延迟等 |
| 队伍槽 | 70 起 | group | 队伍成员的状态信息 |

## 全局固定块（索引 1–20）

以下 20 个块在**所有职业、所有专精**中完全相同：

| 索引 | 名称 | 含义 | 编码方式 |
|---|---|---|---|
| 1 | 锚点 | 绿色标记，Python 定位色块行起点 | 固定 RGB(0, 1/255, 0) |
| 2 | 职业 | 玩家职业 ID | B = 职业ID/255 |
| 3 | 专精 | 当前专精索引 | B = 专精索引/255 |
| 4 | 有效性 | 玩家状态有效性（死亡/坐骑/聊天输入等） | B = 0或1 |
| 5 | 战斗 | 是否在战斗中 | B = 0或1 |
| 6 | 移动 | 是否在移动中 | B = 0或1 |
| 7 | 施法 | 是否正在施法 | B = 0或1 |
| 8 | 引导 | 是否正在引导 | B = 0或1 |
| 9 | 蓄力 | 是否正在蓄力 | B = 0或1 |
| 10 | 蓄力层数 | 蓄力技能的蓄力层数 | B = 层数/255 |
| 11 | 生命值 | 玩家生命值百分比 | B = 百分比(0–1) |
| 12 | 能量值 | 玩家主资源（怒气/能量/法力等）百分比 | B = 百分比(0–1) |
| 13 | 一键辅助 | 一键辅助是否开启 | B = 0或1 |
| 14 | 法术失败 | 最近一次施法是否失败 | B = 0或1 |
| 15 | 目标类型 | 当前目标类型（NPC/玩家/无等） | B = 类型编码/255 |
| 16 | 队伍类型 | 所在队伍类型（无/小队/团队等） | B = 类型编码/255 |
| 17 | 队伍人数 | 队伍成员数量 | B = 人数/255 |
| 18 | 首领战 | 是否在首领战中 | B = 0或1 |
| 19 | 难度 | 副本难度 | B = 难度编码/255 |
| 20 | 英雄天赋 | 当前激活的英雄天赋 | B = 天赋编码/255 |

**编码规则：**
- 布尔值：`B = 0`（假）或 `B = 1`（真）
- 百分比：`B = 实际百分比`，如半血写 `0.5`
- 枚举值：`B = 枚举索引 / 255`，Python 端通过 config.yml 映射还原

## 第二行（计数条行，countBars）：动态分配

### 布局算法

第二行帧 `FuyutsuiCountBars` 位于 TOPLEFT(0, -2)，高度 20px。条从左到右动态分配：

- `nextAvailableIndex` 从 **2** 开始（索引 1 为绿色锚点）
- 每条 bar 消耗 `maxValue + 3` 个像素宽度单位（`maxValue` 值位 + 1 终点标记 + 2 间距）
- 当 `nextAvailableIndex > 255` 时打印警告"空间不足"
- 每个背景像素编码：`R = 1/255`（标记为第二行），`G = 相对索引/255`
- 条之间共享一个 `RGB(200, 200, 200)` 的终点色块
- 切换专精时调用 `ClearAllFuyutsuiBars()` 清空重置

### countBars 的空间占用

以 DK 邪恶为例：
- 天灾打击（maxValue=20）：起始 2，占用背景 1–22，终点 23，下条起始 25
- 腐化（maxValue=3）：起始 25，占用背景 24–28，终点 29，下条起始 31
- 枯萎凋零（maxValue=2）：起始 31，占用背景 30–33，终点 34

总占用：索引 1–34，剩余索引 35–255 空闲。

### Python 端映射

Python 端通过 `config.yml` 中将字段的 `step` 设为 `bar` 来读取：
```yaml
苦修层数: {step: bar, bar: 1, type: "int"}
```
其中 `bar: 1` 表示读取第 1 条 countBar 的值（按声明顺序 1-based）。

## 队伍槽（group）分配规则

### Lua 端定义

每个专精可选定义一个 `type = "group"` 的条目，包含：
- `start`（或所在键）：起始索引
- `num`：每个成员占用槽位数
- 子字段：生命值百分比、职责、驱散、光环状态等

示例（戒律牧师）：
```lua
[70] = {
    type = "group",
    num = 5,
    healthPercent = 1,  -- 子偏移 1：生命值%
    role = 2,           -- 子偏移 2：职责
    dispel = 3,         -- 子偏移 3：驱散
    auras = {
        [4] = { 194384 },              -- 子偏移 4：救赎
        [5] = { 17, 1253593 },         -- 子偏移 5：真言术盾/虚空护盾
    },
}
```

成员 i 的数据从 `start + (i-1) * num` 开始，连续占用 `num` 个槽位。

## 槽位类型汇总

| 类型关键字 | 含义 | 写入方式 | Python 读取方式 |
