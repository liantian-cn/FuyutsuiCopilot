# FuyutsuiCopilot 协作规则

本仓库是 Fuyutsui 的第三方说明文档项目，用来解释如何为 Fuyutsui 写 mod、相关原理、目录结构、配置方式和示例。`Fuyutsui/` 目录是上游项目源码参考，不是本文档项目的编辑目标。

## 基本规则

1. 默认使用 PowerShell 执行命令。
2. Lua 检查使用 `luacheck`。
3. 开始任何编码或文档编辑任务前，先检查 git 工作树；如果存在未提交变更，先创建一个备份提交。
4. 不要修改 `Fuyutsui/` 目录下的任何文件。
5. 只在仓库根目录及根目录下的中文说明目录中新增或修改第三方说明文档。

## 文档组织

1. 每个解释主题单独建立一个中文目录。
2. 每个主题目录内必须有 `readme.md`，作为该主题的主说明文档。
3. 主题目录内可以保存示例代码、配置片段、流程图或其他辅助材料。
4. 根目录的 `readme.md` 只作为目录索引，链接到各个中文主题目录。
5. 文档应面向第三方 mod 作者，重点解释“如何扩展”“为什么这样工作”“需要注意什么”，不要写成 Fuyutsui 源码仓库的维护文档。
6. 不要因为两个主题共享同一条数据链路就合并到同一个目录；只要解释对象不同，就必须拆成不同中文目录。
7. `截图原理` 目录只说明 Python 端如何截图、截图范围、像素格式和 `mss` 相关实现，不写技能冷却、职业决策或配置映射细节。
8. 技能冷却、充能、物品冷却、冷却刷新和职业逻辑如何消费冷却值，必须放在 `技能冷却` 目录，不要写进 `截图原理`。

## 写作约定

1. 优先使用中文说明。
2. 示例应尽量小而完整，避免依赖未解释的上下文。
3. 涉及 Fuyutsui 源码时，只引用路径和行为，不直接修改源码。
4. 如果需要放代码块，优先放在对应主题目录内；短代码可以直接写在 `readme.md` 中。
5. 不确定的行为要标注为待验证，不要猜测成确定结论。

## Fuyutsui介绍

- Fuyutsui是一个魔兽世界辅助工具。
- ./Fuyutsui/是Fuyutsui的官方repo
- Fuyutsui包含插件端和python端。插件端在./Fuyutsui/目录。python端在./Fuyutsui/Fuyutsui/目录下。
- Fuyutsui的插件端将游游戏内的信息显示在屏幕顶端
- Fuyutsui的python端读取插件端产生的像素条、解析数据、产生战斗逻辑、发送按键。
